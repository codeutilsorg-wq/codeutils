About Case Converter

Text case conventions differ across writing styles and programming languages—what is standard in one context is wrong in another. Blog titles use Title Case, Python variables use snake_case, JavaScript functions use camelCase, CSS class names use kebab-case, and constants use CONSTANT_CASE. Manually converting between these formats is tedious and error-prone, especially for multi-word identifiers. Our Case Converter instantly converts any text into any of 10 different case formats with a single click. All processing happens in your browser with no data sent to any server.

Case Formats Reference

  • UPPERCASE: ALL LETTERS CAPITALIZED — for headings that require all caps, constants in some languages
  • lowercase: all letters lowercase — for email addresses, normalized database values
  • Title Case: First Letter Of Each Word Capitalized — for article titles, book titles, UI headings
  • Sentence case: First letter of first word capitalized — for body text, descriptions, labels
  • camelCase: firstWordLowerRestCapitalized — JavaScript variables, JSON keys, TypeScript functions
  • PascalCase: AllWordsCapitalized — JavaScript/TypeScript classes, React components, C# types
  • snake_case: words_separated_by_underscores — Python variables, database column names, file names
  • kebab-case: words-separated-by-hyphens — URL slugs, CSS class names, HTML attributes
  • CONSTANT_CASE: UPPERCASE_WITH_UNDERSCORES — environment variables, constants in Python, Java, C
  • dot.case: words.separated.by.dots — configuration keys, package names, property paths

Common Use Cases

Renaming Variables and Functions

When refactoring code that uses inconsistent naming conventions, or when converting a project from JavaScript to Python, you frequently need to rename identifiers from camelCase to snake_case or vice versa. Paste the identifier, click the target case, and copy the correctly formatted name. This is much faster than manually editing each word boundary, especially for long multi-word names.

Creating URL Slugs and CSS Class Names

Web development requires text in kebab-case for URL paths and CSS class names. Convert blog post titles, product names, or category labels to kebab-case to create clean, web-safe strings ready to use in URLs, HTML attributes, or CSS selectors. The converter handles accented characters and punctuation gracefully, producing clean output.

Formatting Headings and Titles

Writers and content creators frequently need to convert text between sentence case (for most body copy), title case (for headings and article titles), and all-caps (for labels, buttons, and decorative headings). Click the appropriate case button to instantly reformat a heading without manually capitalizing each word.

Tips & Best Practices

  • Prefer kebab-case for URLs and CSS: Google recommends hyphens over underscores in URLs for better SEO, and CSS class names conventionally use kebab-case by most style guides
  • Use snake_case for Python and databases: PEP 8 requires snake_case for variable and function names, and most database naming conventions also prefer snake_case for column names
  • Use camelCase for JavaScript/TypeScript: The dominant convention in JavaScript is camelCase for variables and functions, PascalCase for classes and components—mix them appropriately
  • Use CONSTANT_CASE for environment variables: All environment variables (in .env files, Docker configs, CI/CD settings) follow the convention of SCREAMING_SNAKE_CASE for easy identification

Frequently Asked Questions

Q: Can the converter handle text that is already in a specific case format?

A: Yes. The converter first splits the input text into individual words by detecting boundaries at spaces, underscores, hyphens, and dots—regardless of the current casing. It then reassembles the words in the target format. So "myVariableName", "my-variable-name", and "my_variable_name" all split into the same words before conversion.

Q: Does Title Case capitalize every word including articles?

A: Yes, the Title Case conversion capitalizes the first letter of every word, including small words like "a", "an", "the", "and", "or". For strict AP Style or Chicago Style title case (which lowercases certain prepositions and conjunctions), you would need to manually adjust a few words after conversion.

Q: Is my text private?

A: Yes. All conversion happens in your browser using JavaScript. Your text is never sent to any server, stored, or transmitted anywhere.
